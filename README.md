# Why Greenspun?

I want the simplicity and low-level control of C, Lisp syntax and macros, and the Haskell-inspired typing of Swift in one language.

# FAQs

Rather than enumerate the design decisions behind the language, I'll present them through a fictitious FAQ.

### Why doesn't this "Lisp" have feature X?

I'll counter with this: why don't more languages just use Lisp syntax and reap the low-hanging benefits of easy parsing and macros without feeling the obligation to adopt and implement the entire paradigm? I've been asking myself this question since becoming an obsessive reader of Paul Graham essays. This is what I'm exploring with Greenspun as I see it less as a Lisp dialect and more as C descendant with Lisp syntax. So in regards to feature X, I might implement it in the future, but my two priorities are compile-time macros and runtime evals.
